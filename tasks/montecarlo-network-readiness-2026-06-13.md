# Monte-Carlo over Network Effects (packet loss, random delays) — App Readiness

**Date:** 2026-06-13  
**Question:** how ready is DiaBloS for blocks/studies modeling packet loss, random delays, and Monte Carlo?

## Readiness
 
Frank assessment: DiaBloS today has the SUBSTRATE for a Monte-Carlo network-effects study but cannot run one. None of the three things the user actually needs exist yet: (1) no packet-loss/dropout block, (2) no variable/random transport delay, and (3) no N-run Monte-Carlo manager with per-run reseeding or statistical aggregation. A grep across the whole repo for monte/ensemble/percentile/master_seed/run_index/run_seed found ZERO code matches (only a docs/backlog file). What IS solid: the seeded-RNG pattern in blocks/noise.py (np.random.default_rng(seed), state in params['_rng'], reseed on _init_start_), a correct interpreter routing for path-dependent stochastic blocks (excluded from the compiled fast path), a fixed-step-Euler override for stochastic systems so ensemble members share one timeline, and a clean headless re-simulation primitive (DSim.run_tuning_simulation) that is safe to call N times in a loop. So the engine plumbing is genuinely ready and low-risk to extend, but there is a hard blocker even after blocks are added: reset_memblocks re-seeds every run from the SAME static per-block 'seed' param, so N re-runs produce N IDENTICAL trajectories. Without run-seed injection, Monte Carlo is impossible no matter what blocks exist. Net: roughly 2 small blocks + 1 medium infra piece (run-seed plumbing) gets a working study; full statistical UI/parallelism is more. Estimate: a functional packet-loss + random-delay MC study is a few days of focused work, all built on existing patterns.

## What already exists (substrate to build on)

- Seeded Gaussian Noise source (blocks/noise.py:72-82): np.random.default_rng(seed) built on _init_start_, RNG stored in params['_rng'] per the BaseBlock state-in-params contract; seed!=0 reproducible, seed==0 entropy-seeded. The seed param is present at line 45 (the old doc/params mismatch is fixed). This is the exact RNG/seed template to clone for every new stochastic block.
- Interpolating transport-delay buffer (blocks/transport_delay.py): a deque of (time,value) pairs plus an _interpolate() bisect helper (lines 106-129) that returns the signal at ANY target_time. target_time = time - delay_time at line 71 is the only line that must change to read a per-step/random delay. This buffer+interpolate core is the single most reusable asset for variable/random latency.
- Zero-Order-Hold held-value mechanism (blocks/zero_order_hold.py): _held_value_ + _next_sample_time_ in params, gated by output_only. This is the exact hold-last-value latch a packet-drop block needs (only the periodic schedule must become a probabilistic keep/drop decision).
- Engine pins stochastic systems to fixed-step Euler (simulation_engine.py:1193-1207, stochastic_fns={'Noise'}): guarantees one RNG draw per fixed grid point and a COMMON timeline across runs, which is the prerequisite for elementwise mean/percentile across ensemble members. New stochastic blocks must be registered into this set.
- Correct interpreter routing for path-dependent blocks (system_compiler.py COMPILABLE_BLOCKS, lines ~52-121): Noise/Hysteresis/StateVariable/Impulse and all delay/hold blocks are excluded from the compiled solve_ivp path and run on the interpreter (lib/lib.py execution_loop_headless) where params['_rng'] is honored. Any new packet-loss/variable-delay block inherits this safe routing for free by staying off the allowlist.
- Headless single-run primitive (lib/lib.py DSim.run_tuning_simulation, line 651): re-runs the whole diagram with no UI/dialogs, returns (success,error_msg), sets self.timeline, and leaves scope vectors on each Scope block. Verified safe to call N times in a row — this is the exact loop body a Monte-Carlo runner wraps.
- Proven N-run loop template (lib/engine/optimization_engine.py create_objective ~line 296): set_parameters -> reset_blocks -> execution_init -> execution_batch per evaluation, appending to self.history. Structurally identical to a MC loop (random sampling instead of optimizer picking x; aggregate trajectories instead of scalar cost) — a shared BatchRunner could back both.
- Per-run trace storage + multi-run overlay (lib/services/run_history_service.py record_run line 105, schema {name, y:ndarray, timeline}, arrays defensively copied; modern_ui/widgets/waveform_inspector.py per-run coloring pg.intColor(run_idx) + CSV export). The overlay is the visual basis for spaghetti/envelope plots and reusable as-is.
- Live param-write-then-resim cycle (modern_ui/controllers/tuning_controller.py _execute_tuning line 94): finds a block by name, writes block.params[param] (incl. indexed list elements like denominator[i]), calls run_tuning_simulation, refreshes plots. This is precisely the sample-injection vehicle a sweep/MC loop performs, minus a debounce timer.
- Deterministic PRBS LFSR (blocks/prbs.py): seeded, reproducible, intentionally treated as non-stochastic by the engine. Useful as a fixed binary stimulus but NOT a substitute for Bernoulli packet loss.

## Key gaps

- No run-scoped seed plumbing (HARD BLOCKER). reset_memblocks (simulation_engine.py:713-733) re-seeds Noise/PRBS from the SAME static per-block 'seed' param every run, so N re-runs yield N IDENTICAL trajectories. There is no master_seed/run_index/seed_offset concept anywhere in the code (grep: zero matches). Without injecting a per-run derived sub-seed, Monte Carlo is impossible even after the blocks exist.
- No packet-loss / dropout block. Nothing draws a per-sample Bernoulli(p) keep/drop decision and on a drop holds the last received value (or outputs zero/NaN). Can only be hand-wired from Noise+CompareToConstant+Switch, where the drop rate p is an opaque nonlinear function of (mu,sigma,constant) and the cadence is wrong — clumsy, hard to seed, not discoverable.
- No variable / input-driven transport delay. blocks/transport_delay.py reads delay_time as a STATIC param with no input port, so time-varying/random latency and jitter cannot be expressed at all. This is the single biggest missing piece for random-delay studies (the interpolation core already supports it; only target_time and buffer pruning need to change).
- No direct Bernoulli/Uniform random source. Only Gaussian Noise and deterministic PRBS exist; neither emits a per-sample 0/1 keep/drop flag with a tunable probability p, nor a clamped U[a,b] draw to scale random delays/jitter.
- No Monte-Carlo / batch manager with statistical aggregation. run_tuning_simulation re-runs one config at a time and RunHistoryService stores traces (limit=5), but nothing loops N randomized runs with per-run seeds, and nothing computes mean/std/min-max envelope/percentile bands across runs or per-run scalar metrics (overshoot, settling time, ISE, divergence flag).
- RunHistoryService limit=5 (line 13) would evict MC members. N=100..1000 runs need a separate batch-tagged ensemble container, not the 5-entry interactive rolling buffer (which also persists to JSON).
- Variable-delay buffer pruning hazard. TransportDelay prunes history at 2x delay_time (line 96). With a random delay that occasionally spikes, the history needed for a large target_time may already be pruned, silently returning a stale/initial value. A variable-delay block must size its buffer to the MAXIMUM possible delay.
- Two latent reproducibility bugs (dead today, must fix before any compiled-Noise change). Unseeded global np.random.randn() at system_compiler.py:829 (exec_noise) and simulation_engine.py:1732 (compiled-replay) ignore params['_rng']. Harmless now because Noise is non-compilable, but they would silently corrupt MC reproducibility the moment Noise is added to COMPILABLE_BLOCKS (a change the backlog proposes).
- No bursty/correlated loss model (Gilbert-Elliott two-state Markov). Realistic networked control cares about correlated dropouts, not just i.i.d. Bernoulli. No block models this.
- No jittered/event-driven sampler. All holds/samplers (ZOH, FOH, RateTransition) fire on strictly periodic schedules; none can randomize the sample instant.
- No parallel execution. MC over hundreds of interpreter-pinned fixed-step Euler runs is slow serially; only multiprocessing.freeze_support exists. No worker path that rebuilds the engine from a serialized .diablos and applies seed/param overrides.

## Recommended new blocks

- **PacketLoss / Dropout block** _(effort: small)_ — 1-in/1-out block: at each sample instant draw keep/drop ~ Bernoulli(loss_prob) from a seeded RNG; on keep pass input, on drop output held last value (drop_mode {hold,zero,nan}). Params: loss_prob, sample_time, seed, drop_mode, initial_value — all state (_held_value_, _rng, _next_sample_time_) in params per the engine rule. Mark b_type=1 (memory block, like Delay) so it can sit in feedback loops; keep OFF COMPILABLE_BLOCKS and register its fn into simulation_engine stochastic_fns:1197. Run tools/sync_block_registry.py after. CORE deliverable for packet loss.
- **VariableTransportDelay block (delay-input port)** _(effort: small)_ — Clone TransportDelay with a 2nd input port 'tau' (or min_delay/max_delay params). Reuse the deque + _interpolate() (lines 106-129) verbatim; change only target_time = time - clamp(tau_input, 0, max_delay). Add a max_delay param and prune the buffer to max_delay instead of the hardcoded 2x delay_time (line 96), and clamp/extrapolate gracefully when a request predates the buffer. Keep OFF the compiled allowlist. Delivers time-varying/random network latency directly. CORE deliverable for random delay.
- **Bernoulli / Uniform random source block** _(effort: small)_ — Sources block emitting a per-sample Bernoulli(p) 0/1 draw, a U[a,b] draw, or a random integer, with seed semantics identical to Noise (default_rng in params['_rng'], seed==0 entropy). Drives a Switch ctrl port for explicit packet gating and supplies clamped random delay/jitter values for VariableTransportDelay. Direct clone of blocks/noise.py:72-82; auto-excluded from compile; add to stochastic_fns. Useful standalone primitive even if PacketLoss embeds its own RNG.
- **NetworkChannel composite (loss + variable delay + optional jitter)** _(effort: medium)_ — Single 'communication link' block combining the PacketLoss gate and the VariableTransportDelay (uniform/normal/exponential, clamped to [min,max]) plus optional sampling jitter, all reproducibly seeded. Implement as an internal subsystem composing the two new blocks (blocks/subsystem.py) so users drop ONE block between controller and plant — the idiomatic Simulink-style authoring model. Also interpreter-routed.
- **Gilbert-Elliott correlated-loss option** _(effort: small)_ — Two-state Markov loss (good/bad with p_good->bad, p_bad->good) as a mode on PacketLoss or a standalone loss-flag source. Supplies the correlated/bursty dropout model that i.i.d. Bernoulli lacks — the realistic channel for networked-control research. Same seeded-RNG pattern; b_type per its role.
- **Jittered sampler option on ZOH / RateTransition** _(effort: small)_ — Optional params (jitter_std or jitter_dist + seed) that perturb _next_sample_time_ when advancing it, modeling transmission-instant jitter. Localized change to the scheduling lines (zero_order_hold.py:82-89; rate_transition.py:213-214,257-258); default jitter=0 preserves all current behavior/tests.

## Recommended infrastructure

- **Run-seed plumbing (master_seed + run_index -> per-block sub-seed)** _(effort: medium)_ — THE hard blocker. Before each run (in/around reset_memblocks, simulation_engine.py:713-733), when a block exposes a 'seed' param and the run is part of an experiment, derive a deterministic sub-seed via np.random.SeedSequence([master_seed, run_index, block_hash]) and write it into exec_params['seed'] so the existing _init_start_ reseed in noise.py/prbs.py/new blocks picks it up. Makes each run different yet the whole experiment bit-reproducible from one master seed. Without this, all MC runs are identical. Leverages: noise.py:72-82 seed-from-param, reset_memblocks already flips _init_start_ every run — only seed injection is missing.
- **BatchRunner / re-simulation manager (shared core for MC + sweep)** _(effort: medium)_ — Headless orchestrator: given a sample/seed generator and N draws, calls dsim.run_tuning_simulation (lib/lib.py:651) once per draw, harvests scope_plotter.get_scope_traces (line 875) after each, and stores (timeline, traces, metrics) into a DEDICATED ensemble container — NOT the 5-entry run_history (run_history_service.py:13). Factor the per-run loop currently inlined in OptimizationEngine.create_objective (line 296) into this so optimizer, sweep, and MC share one driver. Optional progress/cancel callback; run on a background QThread. Reuses tuning_controller _execute_tuning param-write logic (line 94) for applying overrides.
- **MonteCarloManager (seed injection + statistics) on top of BatchRunner** _(effort: medium)_ — User marks params uncertain (nominal +/- tol or distribution) and/or enables seed randomization; manager derives per-block seeds per run from one master seed via the run-seed plumbing, runs N sims, computes ensemble stats: elementwise mean/std/percentile bands over the shared timeline, plus per-run scalar metrics (overshoot, settling time, ISE, final value, divergence flag) with histograms. The fixed-step-Euler-on-stochastic detection (simulation_engine.py:1193-1207) guarantees the common timeline these statistics require. Emits an EnsembleResult the inspector renders.
- **Envelope + histogram rendering in WaveformInspector** _(effort: medium)_ — Add an 'Ensemble' view: faint per-run spaghetti, a bold mean curve, and a shaded percentile band (pyqtgraph FillBetweenItem) over the shared timeline, plus a histogram/box panel for a selected scalar metric. Aggregate so 500 members render as a band, not 500 legend entries. Reuses existing per-run coloring (waveform_inspector.py:182), scrub readout, CSV export (line 262); pyqtgraph already a dependency.
- **Route the two dead Noise executors through the seeded RNG** _(effort: medium)_ — Replace unseeded np.random.randn() in system_compiler.py:829 (exec_noise) and simulation_engine.py:1732 (compiled-replay) with draws from params['_rng'] matching noise.py:76. Currently dead (Noise non-compilable) but a latent reproducibility bug that would silently corrupt MC the moment Noise joins COMPILABLE_BLOCKS. Cheap insurance; do it before any compiled-Noise work.
- **Multiprocessing parallel batch execution** _(effort: large)_ — Distribute BatchRunner's independent runs across cores: serialize the diagram via FileService, ship seed/param overrides to workers that rebuild the engine, run headless, return timeline+vectors for aggregation. High value because stochastic MC is pinned to slower fixed-step-Euler interpreter runs. Leverages multiprocessing.freeze_support already in diablos_modern.py and FileService .diablos round-trip. Defer until N is large.

## Minimum viable path

Smallest ordered sequence to a working packet-loss + random-delay Monte-Carlo study, each step reusing existing infrastructure: (1) Run-seed plumbing FIRST (medium) — inject master_seed+run_index -> per-block sub-seed in/around reset_memblocks (simulation_engine.py:713-733) writing exec_params['seed']; without this every run is identical and nothing else matters. (2) PacketLoss block (small) — clone the noise.py:72-82 seeded-RNG pattern + zero_order_hold.py held-value latch; b_type=1; keep off COMPILABLE_BLOCKS; add 'PacketLoss' to stochastic_fns at simulation_engine.py:1197; run tools/sync_block_registry.py. (3) VariableTransportDelay block (small) — copy transport_delay.py, change target_time to use a tau input port clamped to [0,max_delay], and prune to max_delay instead of 2x (fixes the line-96 stale-value hazard); off the compiled allowlist. (4) Minimal BatchRunner (medium) — a headless loop calling dsim.run_tuning_simulation (lib/lib.py:651) N times, incrementing run_index each iteration, harvesting scope_plotter.get_scope_traces (line 875) into a list (bypass the 5-entry run_history). Model the loop on OptimizationEngine.create_objective (optimization_engine.py:296). (5) Basic aggregation (part of step 4, medium) — elementwise mean and 5/95 percentile bands across the shared fixed-step-Euler timeline (guaranteed common by simulation_engine.py:1193-1207), printed/returned even before any UI. At this point the user can wire Source -> [PacketLoss] -> [VariableTransportDelay driven by a uniform/noise source] -> plant, run N seeded sims, and read mean+envelope of the output — a complete packet-loss + random-delay MC study. DEFER to later: NetworkChannel composite, Gilbert-Elliott, jittered samplers, the WaveformInspector envelope/histogram UI, multiprocessing, and the dead-Noise-executor fix (only needed if Noise is ever re-admitted to the compiled path).
